# ADR (Architecture Decision Records)

**되돌리기 비싼 결정**이 실제로 생겼을 때만 한 장 적는다. 미리 줄세우지 않는다(과설계 방지).

- 파일명: `NNNN-제목.md` (4자리, `0001`부터)
- 한번 "채택"된 ADR은 고치지 않는다. 결정이 바뀌면 **새 ADR로 대체(supersede)**하고, 현재 규칙은 `AGENTS.md`/`docs/`를 갱신한다.
- ADR엔 *왜*를 적고, *현재 규칙*은 `AGENTS.md`/`docs/`에 둔다(이유를 양쪽에 복붙하지 않는다).

## 템플릿

```markdown
# NNNN. <결정 제목>

- 상태: 채택 | 대체됨(→NNNN)
- 날짜: YYYY-MM-DD

## 맥락
무엇을 정해야 했고 어떤 제약이 있었나.

## 결정
무엇으로 정했나.

## 기각한 대안
검토했지만 안 고른 것 + 이유.

## 결과 / 트레이드오프
이 결정으로 생기는 이득과 비용, 되돌릴 때의 비용.
```
